# StyleSeed-Inspired Design Skill

This local skill is a Codex-compatible design quality gate inspired by the public StyleSeed project.

Source reference: https://github.com/bitjaru/styleseed

## Goal

Prevent generic AI-looking UI. Build interfaces that look intentionally designed, coherent, premium, and usable.

## Core rules

1. Use one primary accent color. Everything else should be grayscale or semantic state color.
2. Do not use pure black as the default surface. Use refined near-black or controlled dark neutrals.
3. Keep one radius personality across the whole UI.
4. Keep one shadow/elevation language across the whole UI.
5. Keep one icon style across the whole UI.
6. Use an 8px spacing rhythm.
7. Make one focal point per screen.
8. Do not repeat identical card patterns over and over.
9. Avoid the generic AI pattern of pale icon chips on every feature card.
10. Use real empty, loading, success, warning, and error states.
11. Use accessible contrast and visible focus states.
12. Make mobile layout intentional, not just stacked desktop content.
13. Use typography hierarchy: display, heading, body, label, caption.
14. Make the CTA obvious and emotionally confident.
15. Run the quality gate before final output.

## Design lock

Before building a new UI, define or update a design lock:

- Product type
- Target user
- Surface: mobile app, web app, dashboard, landing page, admin panel, etc.
- Mood: calm, sharp, luxury, playful, technical, cinematic, minimal, bold
- Accent color
- Font direction
- Radius style
- Motion style
- Density: compact, balanced, airy

If the user gives no brand, pick a serious default instead of random purple/blue gradients.

## Quality gate

Before finishing, check:

- First impression feels premium.
- Product purpose is clear in 3 seconds.
- Primary CTA is visible.
- Layout has strong hierarchy.
- Cards are aligned and spaced consistently.
- Color usage is controlled.
- Text is readable.
- Mobile version is polished.
- Components include hover/focus/disabled/loading states.
- No obvious template sections.
- No decorative overload.

If any item fails, improve the UI before responding.
