# TestWeb — Codex Design Skills

This repository has three local Codex-compatible design skills in `.agents/skills/`:

1. `styleseed` — premium UI quality gate and anti-generic design rules inspired by StyleSeed.
2. `ui-ux-pro-max` — UI/UX design-system intelligence inspired by UI/UX Pro Max.
3. `ui-design` — Codex-oriented production frontend design workflow inspired by Codex Design Skill.

When working on UI, websites, landing pages, dashboards, app screens, components, or redesigns in this repo:

- Use `ui-design` for the implementation workflow.
- Use `ui-ux-pro-max` for design-system choices, accessibility, typography, colors, layout, motion, and UX validation.
- Use `styleseed` to prevent generic AI-looking UI and to run a strict visual quality gate.

Default design direction:

- premium, modern, clean, cinematic, futuristic;
- mobile-first and responsive;
- strong visual hierarchy;
- one primary accent color unless a real brand system exists;
- polished hover, focus, loading, empty, and error states;
- no default-looking forms, cards, buttons, navbars, or tables;
- no random gradients, emoji icons, or repeated generic icon cards.

Before shipping UI changes, run a visual QA pass:

- Is the purpose clear in the first screen?
- Is the primary action obvious?
- Is spacing consistent?
- Is mobile intentionally designed?
- Is contrast readable?
- Are interactive states present?
- Does the result look custom and premium rather than AI-generated?
